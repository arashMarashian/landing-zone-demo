Azure Landing Zone Concepts I Learned While Building This Project
  1. App Service Deployment Architecture
    • How Azure App Service handles Linux-based web apps
    • Understanding deployment slots (Production vs Staging)
    • Difference between ZIP Deploy, OneDeploy, and Oryx build pipeline
    • How SCM (Kudu) manages app packages and runtime execution
  2. Identity & Access Management (IAM) in Azure
    • Creating and using User-Assigned Managed Identity
    • Assigning appropriate RBAC roles (Website Contributor, Reader, Owner)
    • Understanding role scope (Resource vs Resource Group vs Subscription)
    • Using Managed Identity inside GitHub Actions (federated credentials)
  3. Azure Landing Zone Governance Concepts
    • Azure Policy impact during resource creation
    • Required tags (Owner, Environment, Project) for compliance
    • Naming standards and resource hierarchy considerations
    • Understanding Subscription-level restrictions on deployments
  4. Networking Fundamentals in an LZ
    • Enabling VNet integration for App Service
    • Understanding how outbound traffic works from App Service
    • Using /health endpoints for probes and network diagnostics
    • Differences between private and public App Service access
  5. Application Configuration & Secret Management
    • Storing secrets (like API keys) in App Settings
    • Using environment variables inside your app (APP_MODE, REGION, LOG_LEVEL, AI_API_KEY)
    • Separating code from configuration â€“ a core LZ best practice
    • Secure handling of third-party API keys (OpenAI, Google Gemini, Groq)
  6. CI/CD Pipeline with GitHub Actions (Azure Standard)
    • Building a Python app using GitHub Actions
    • Authenticating GitHub â†’ Azure using OIDC (OpenID Connect)
    • Creating artifacts, packaging, and deploying through pipelines
    • Debugging CI/CD failures (409 conflicts, build timeouts, permission errors)
  7. Observability & Monitoring Basics
    • Using Azure Deployment Center logs
    • Understanding App Service build logs (Oryx)
    • Using /health route for monitoring and probes
    • Troubleshooting internal server errors and platform-level failures
  8. Azure Resource Organization Principles
    • Importance of Resource Groups in Landing Zones
    • Consistency in resource naming conventions
    • Applying LZ governance rules while deploying workloads
    • Understanding the impact of subscription-level tagging policies
  9. Cloud Application Architecture (Landing Zone Perspective)
    • Designing apps to be cloud-friendly (stateless, config-driven, restart-safe)
    • Decoupling application from underlying infrastructure
    • Handling dynamic configurations through environment variables
    • Understanding how the platform deploys, restarts, and scales apps
  10. Real-World Deployment Troubleshooting
    • Oryx build freezes
    • OneDeploy 409 conflicts
    • Identity permission issues
    • Tag policy failures
    • Long deployment times
    • API quota & authentication errors (OpenAI / Gemini / Groq)
