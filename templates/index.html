Landing Zone concepts demonstrated
This single web app touches multiple Azure Landing Zone pillars: deployment, identity, governance, networking, configuration, and CI/CD.

  • App Service deployment on Linux with Oryx & OneDeploy.
  • User-Assigned Managed Identity & RBAC role assignment.
  • Azure Policy & mandatory tagging for compliant resources.
  • VNet integration patterns for App Service workloads.
  • Configuration via App Settings & environment variables.
  • GitHub Actions CI/CD with OIDC trust to Azure.
  • Health endpoints and basic observability for the app.
  • Organizing resources using Landing Zone naming & RG structure.
  • Troubleshooting real deployment issues (409, identity, policy, API).
